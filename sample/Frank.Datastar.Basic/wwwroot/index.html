RESTful Resource Patterns with Hypermedia-First Updates

Key Principles

  • Resource URLs: Use nouns, not verbs (/contacts/1, not /getContact)
  • HTTP Methods: GET=retrieve, PUT=update, DELETE=remove, POST=create
  • Query Params: Use for filtering (/fruits?q=apple)
  • Hypermedia First: Server sends HTML, client renders it
Jump to: Click-to-Edit | Search | Delete | Bulk Update | Form Validation

Click-to-Edit GET/PUT Resource

Edit a contact inline using proper REST: GET establishes SSE, edit/save are fire-and-forget.

Click "Load Contact" to establish SSE connection. Then Edit/Save use that channel.

Search with Filtering GET + Query Params

Filter a collection using GET with query parameters. GET /fruits establishes SSE, subsequent searches are fire-and-forget.

Click "Load Fruits" first to establish SSE, then type to search.

Row Deletion DELETE Resource

Delete items using HTTP DELETE. GET /items establishes SSE, DELETE is fire-and-forget.

Click "Load Items" to establish SSE. Delete buttons use that channel.

Bulk Update PUT Collection

Update multiple resources using PUT. GET /users establishes SSE, bulk updates are fire-and-forget.

Click "Load Users" to establish SSE. Select users and click Activate/Deactivate.

Form Validation POST with Validation

Real-time validation. GET /registrations/form establishes SSE, validation and submit are fire-and-forget.

Click "Show Form" to establish SSE. Validation happens as you type.

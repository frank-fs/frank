# Work Packages: Frank.Provenance (PROV-O State Change Tracking)

**Inputs**: Design documents from `kitty-specs/006-prov-o-state-change-tracking/` (spec, plan, research, data-model, quickstart)
**Prerequisites**: plan.md (required), spec.md, research.md, data-model.md
**GitHub Issue**: #77 | **Parent**: #80 (Semantic Metadata-Augmented Resources)
**Dependencies**: Frank.Statecharts (#87, complete), Frank.LinkedData (#75, complete)

**Tests**: Include tests for all public API surface. Frank convention: Expecto + TestHost.

**Organization**: Fine-grained subtasks (`Txxx`) roll up into work packages (`WPxx`). Each work package must be independently deliverable and testable.

**Prompt Files**: Each work package references a matching prompt file in `kitty-specs/006-prov-o-state-change-tracking/tasks/` generated by `/spec-kitty.tasks`.

## Work Package WP02: IProvenanceStore + MailboxProcessorStore (Priority: P0)

**Goal**: Define `IProvenanceStore` interface and implement `MailboxProcessorProvenanceStore` with serialized writes, indexed queries, configurable retention policy, and proper disposal.
**Independent Test**: Store appends records, queries by resource/agent/time range return correct subsets, retention evicts oldest records, disposal drains queue.
**Prompt**: `tasks/WP02-provenance-store.md`
**Estimated Size**: ~450 lines
**Requirement Refs**: FR-007, FR-008, FR-009, FR-015

### Included Subtasks
- [ ] T007 Create `src/Frank.Provenance/Store.fs` with `IProvenanceStore` interface
- [ ] T008 Create `src/Frank.Provenance/MailboxProcessorStore.fs` with `StoreMessage` DU, `StoreState`, and `MailboxProcessorProvenanceStore` class
- [ ] T009 Implement retention policy (oldest-first eviction in configurable batches)
- [ ] T010 Implement `IDisposable` with drain of pending appends and graceful post-disposal behavior
- [ ] T011 Create `test/Frank.Provenance.Tests/StoreTests.fs` with append, query, retention, disposal, and concurrency tests

### Dependencies
- Depends on WP01 (core types)

### Risks & Mitigations
- Index rebuild cost on eviction: amortized by batch eviction (default 100). Test at 10K records to verify sub-millisecond performance.
- `ObjectDisposedException` after disposal: post-disposal messages are no-ops (logged, not thrown).

## Work Package WP03: GraphBuilder (dotNetRdf) (Priority: P1)

**Goal**: Implement `GraphBuilder` module that projects `ProvenanceRecord list` into a dotNetRdf `IGraph` with complete PROV-O triples, namespace prefixes, and typed literals.
**Independent Test**: Given sample `ProvenanceRecord` instances, `toGraph` produces an `IGraph` with correct triples for Activity, Agent, Entity, and all PROV-O relationships.
**Prompt**: `tasks/WP03-graph-builder.md`
**Estimated Size**: ~400 lines
**Requirement Refs**: FR-004, FR-005, FR-006, FR-014

### Included Subtasks
- [ ] T012 Create `src/Frank.Provenance/GraphBuilder.fs` with `toGraph : ProvenanceRecord list -> IGraph`
- [ ] T013 Implement Activity triple construction (type, startedAtTime, endedAtTime, wasAssociatedWith, used, frank:httpMethod, frank:eventName). Note: must include explicit `prov:wasAssociatedWith` triple construction linking activities to agents (FR-006).
- [ ] T014 Implement Agent triple construction (type based on AgentType DU, prov:label)
- [ ] T015 Implement Entity triple construction (type, wasGeneratedBy, wasAttributedTo, wasDerivedFrom, frank:stateName)
- [ ] T016 Create `test/Frank.Provenance.Tests/GraphBuilderTests.fs` with triple validation tests

### Dependencies
- Depends on WP01 (types + vocabulary)

### Risks & Mitigations
- dotNetRdf API version: pin to same version as Frank.LinkedData (3.5.1). Verify `UriFactory.Root.Create` is available.
- LlmAgent subclass triple: requires both `prov:SoftwareAgent` type and `frank:LlmAgent` annotation on the same node. Test dual-typing explicitly.

## Work Package WP04: Provenance Middleware (Content Negotiation) (Priority: P1)

**Goal**: Implement middleware that intercepts GET requests with `application/vnd.frank.provenance+*` Accept headers, queries the store, builds an RDF graph, and serializes the response using Frank.LinkedData's `writeRdf`.
**Independent Test**: GET with provenance Accept header returns valid Turtle/JSON-LD/RDF-XML; standard Accept headers pass through to normal handler.
**Prompt**: `tasks/WP04-provenance-middleware.md`
**Estimated Size**: ~450 lines
**Requirement Refs**: FR-010

### Included Subtasks
- [ ] T017 Create `src/Frank.Provenance/Middleware.fs` with Accept header parsing for `vnd.frank.provenance+*` media types
- [ ] T018 Implement media type mapping (`vnd.frank.provenance+turtle` -> `text/turtle`, etc.) and response serialization
- [ ] T019 Implement resource URI extraction from request path and store query
- [ ] T020 Implement empty graph handling (200 with empty collection, not 404)
- [ ] T021 Create `test/Frank.Provenance.Tests/MiddlewareTests.fs` with TestHost integration tests

### Dependencies
- Depends on WP02 (store for querying), WP03 (graph builder for serialization)

### Risks & Mitigations
- Middleware ordering with Frank.LinkedData: provenance middleware must run before LinkedData middleware to intercept `vnd.frank.provenance+*` types first.
- Frank.LinkedData's `writeRdf` may not be public: verify accessibility; if internal, add `InternalsVisibleTo` or use dotNetRdf writers directly.

## Work Package WP05: TransitionObserver (Priority: P1)

**Goal**: Implement `TransitionObserver` that subscribes to `onTransition` hooks from Frank.Statecharts, extracts agent identity from `ClaimsPrincipal`, constructs `ProvenanceRecord` instances, and appends them to `IProvenanceStore`.
**Independent Test**: Given a mock `TransitionEvent`, the observer produces a correct `ProvenanceRecord` with proper agent classification and appends it to the store.
**Prompt**: `tasks/WP05-transition-observer.md`
**Estimated Size**: ~400 lines
**Requirement Refs**: FR-001, FR-002, FR-003, FR-004, FR-005, FR-006, FR-013

### Included Subtasks
- [ ] T022 Create `src/Frank.Provenance/TransitionObserver.fs` with `IObserver<TransitionEvent>` implementation
- [ ] T023 Implement agent extraction from `ClaimsPrincipal` (Person, SoftwareAgent, LlmAgent classification)
- [ ] T024 Implement `ProvenanceRecord` construction from `TransitionEvent` (Activity, pre/post Entities, Agent)
- [ ] T025 Implement resilient error handling (OnError, ObjectDisposedException logged but not propagated)
- [ ] T026 Create `test/Frank.Provenance.Tests/TransitionObserverTests.fs` with agent classification and record creation tests

### Dependencies
- Depends on WP01 (types), WP02 (store for appending)

### Risks & Mitigations
- Frank.Statecharts `TransitionEvent` type may not exist yet: define expected shape in research.md; adapt when Statecharts API is finalized.
- `X-Agent-Type` header not available in `TransitionEvent`: may need to be threaded through from `HttpContext`. Document the dependency on Statecharts providing `User` and HTTP metadata.

**Independent Test**: Trigger transitions from authenticated user, unauthenticated request, and LLM-origin request; verify each produces correctly typed agent in both store records and RDF output.
**Prompt**: `tasks/WP06-agent-type-discrimination.md`
**Estimated Size**: ~350 lines
**Requirement Refs**: FR-003, FR-012

### Included Subtasks
- [ ] T027 Implement `X-Agent-Type` and `X-Agent-Model` header extraction in TransitionObserver
- [ ] T028 Implement `AgentType.LlmAgent` graph builder support (dual `prov:SoftwareAgent` + `frank:LlmAgent` typing)
- [ ] T029 LLM-specific agent detection (X-Agent-Type/X-Agent-Model header extraction, extends WP05 base agent extraction)
- [ ] T030 Create `test/Frank.Provenance.Tests/AgentTypeTests.fs` with discrimination tests for all three agent types

### Dependencies
- Depends on WP03 (graph builder), WP05 (observer)

### Risks & Mitigations

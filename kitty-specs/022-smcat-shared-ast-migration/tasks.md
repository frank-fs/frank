## Work Package WP02: Parser Migration (Priority: P0)

**Goal**: Migrate `Parser.fs` to produce `Ast.ParseResult` with `StatechartDocument` directly, eliminating the need for `Mapper.fs`. The parser's internal state, element construction, and public API all switch to shared AST types.
**Independent Test**: `parseSmcat "idle => active: start;"` returns an `Ast.ParseResult` with correct `StatechartDocument` content.
**Prompt**: `/tasks/WP02-parser-migration.md`
**Estimated Prompt Size**: ~500 lines

### Included Subtasks
- [ ] T004 Update `ParserState` to use `Ast.StatechartElement list`, `Ast.ParseFailure list`, `Ast.ParseWarning list`
- [ ] T005 Update `parseStateDeclaration` to construct `Ast.StateNode` directly with `SmcatAnnotation` for color/label attributes and `Ast.StateActivities` for activities
- [ ] T006 Update `parseTransition` to construct `Ast.TransitionEdge` directly with label components split into `Event`/`Guard`/`Action` fields
- [ ] T007 Update `parseDocument` to return `Ast.StatechartDocument` (was `SmcatDocument`)
- [ ] T008 Update `addError`/`addWarning` helper functions to use `Ast.ParseFailure`/`Ast.ParseWarning` (with `SourcePosition option`)
- [ ] T009 Update `parse` and `parseSmcat` public API to return `Ast.ParseResult`
- [ ] T010 Update composite state handling: `StateNode.Children` is `StateNode list` (not `SmcatDocument option`)

### Implementation Notes
- The Mapper.fs logic for converting `SmcatState` -> `StateNode`, `SmcatTransition` -> `TransitionEdge`, etc. is inlined directly into the parser's element construction
- Key mapping: `SmcatAttribute` with key="color" -> `SmcatAnnotation(SmcatColor value)`, key="label" -> `SmcatAnnotation(SmcatStateLabel value)`, other keys -> `SmcatAnnotation(SmcatActivity(key, value))`
- `StateActivity` (single option per kind) -> `StateActivities` (list per kind): use `Option.map List.singleton |> Option.defaultValue []`
- Source positions change from `SourcePosition` (always present) to `Ast.SourcePosition option` (wrapped in `Some`)
- Composite state children: previously `SmcatDocument option`, now `StateNode list` (need to filter only `StateDecl` elements and extract the `StateNode`; transitions within composites become top-level elements of the child)

### Parallel Opportunities
- None within this WP -- changes are interdependent within `Parser.fs`

### Dependencies
- Depends on WP01 (Types.fs changes must be in place).

### Risks & Mitigations
- Composite state children handling is the most complex change: `StateNode.Children` is `StateNode list` but composites also contain transitions. The parser must handle this by building both child states and child transitions, storing states in `Children` and transitions as `TransitionElement` entries in the parent document's element list. Follow the Mapper.fs `toChildStateNodes` pattern.

## Work Package WP03: Serializer and Generator (Priority: P1)

**Goal**: Create the new `Serializer.fs` (AST -> smcat text) and refactor `Generator.fs` (metadata -> AST), completing the bidirectional shared AST pipeline. Delete `Mapper.fs`.
**Independent Test**: `Serializer.serialize` produces valid smcat text from a hand-constructed `StatechartDocument`; `Generator.generate` returns `Ok(StatechartDocument)` from valid `StateMachineMetadata`.
**Prompt**: `/tasks/WP03-serializer-generator.md`
**Estimated Prompt Size**: ~500 lines

### Included Subtasks
- [ ] T011 [P] Create `src/Frank.Statecharts/Smcat/Serializer.fs` with `serialize: StatechartDocument -> string` function
- [ ] T012 [P] Refactor `src/Frank.Statecharts/Smcat/Generator.fs` to return `Result<StatechartDocument, GeneratorError>` from `StateMachineMetadata`, following `Wsd.Generator` pattern
- [ ] T013 Delete `src/Frank.Statecharts/Smcat/Mapper.fs` from the filesystem

### Implementation Notes
- **Serializer** reads `SmcatAnnotation` entries from `StateNode.Annotations` for color/label attributes; reads `StateNode.Activities` for entry/exit/do; handles composite states recursively via `StateNode.Children`; uses `needsQuoting`/`quoteName` helpers (smcat allows dots/hyphens unlike WSD)
- **Generator** follows `Wsd.Generator` exactly: validate boxed Machine type, extract states from `StateHandlerMap`, order (initial first, alphabetical), create `StateDecl` + `TransitionElement` entries, return `StatechartDocument`; define `GeneratorError = UnrecognizedMachineType of typeName: string`
- The `formatLabel`, `formatTransition`, `generateTo` functions are deleted from Generator.fs (their output logic moves to Serializer.fs)

### Parallel Opportunities
- T011 (Serializer) and T012 (Generator) can be developed in parallel -- they are independent files

### Dependencies
- Depends on WP01 and WP02 (Parser must produce shared AST; Types.fs must be clean).

### Risks & Mitigations
- Serializer must handle `StatechartDocument` with no `SmcatAnnotation` entries gracefully (e.g., documents from other format parsers). Use sensible defaults: no attributes, no activities, Regular state type.
- Round-trip fidelity: serializer output must be re-parseable. Validate with manual testing before moving to WP04.

## Work Package WP04: Test Suite Migration (Priority: P1)

**Goal**: Update all smcat test files to use shared AST types. All existing test scenarios must pass with the migrated parser, serializer, and generator.
**Independent Test**: `dotnet test` passes for all smcat tests.
**Prompt**: `/tasks/WP04-test-suite-migration.md`
**Estimated Prompt Size**: ~500 lines

### Included Subtasks
- [ ] T014 [P] Update `test/Frank.Statecharts.Tests/Smcat/ParserTests.fs`: replace `SmcatDocument`/`SmcatState`/`SmcatTransition` assertions with `StatechartDocument`/`StateNode`/`TransitionEdge`; replace `StateDeclaration s` with `Ast.StateDecl s`; replace `s.Name` with `s.Identifier`, `s.StateType` with `s.Kind`; replace `t.Label.Value.Event` with `t.Event`
- [ ] T015 [P] Update `test/Frank.Statecharts.Tests/Smcat/ErrorTests.fs`: replace smcat-local `ParseResult`/`ParseFailure`/`ParseWarning` with `Ast.*` types; update `Position` from value to `Position.Value` (option unwrap)
- [ ] T016 [P] Update `test/Frank.Statecharts.Tests/Smcat/LabelParserTests.fs`: update `SourcePosition` reference to `Ast.SourcePosition` for the `pos` helper
- [ ] T017 [P] Update `test/Frank.Statecharts.Tests/Smcat/GeneratorTests.fs`: generator now returns `Result<StatechartDocument, GeneratorError>` -- unwrap with `Result.get` or pattern match; assert against `StatechartDocument` structure instead of raw text
- [ ] T018 [P] Update `test/Frank.Statecharts.Tests/Smcat/RoundTripTests.fs`: replace `generateFromDocument` with `Serializer.serialize`; replace `SmcatDocument` types with `StatechartDocument`; replace `StateType` with `StateKind`; replace `StateDeclaration`/`TransitionElement` pattern matches with `Ast.StateDecl`/`Ast.TransitionElement`

### Implementation Notes
- Most changes are mechanical find-and-replace
- Key pattern replacements:
  - `open Frank.Statecharts.Smcat.Types` -> add `open Frank.Statecharts.Ast`
  - `StateDeclaration s` -> `StateDecl s`
  - `TransitionElement t` -> `Ast.TransitionElement t` (or unqualified if `Ast` is opened after `Types`)
  - `s.Name` -> `s.Identifier`
  - `s.StateType` -> `s.Kind`
  - `t.Label.Value.Event` -> `t.Event` (label components now on `TransitionEdge` directly)
  - `t.Label None` -> label info now split into `Event = None`, `Guard = None`, `Action = None`
  - `t.Target` -> `t.Target.Value` (now `string option`)
  - `s.Children.Value.Elements` -> navigating `StateNode.Children` list
  - `err.Position.Line` -> `err.Position.Value.Line` (now `SourcePosition option`)
- LexerTests.fs and LabelParserTests.fs should need minimal changes (lexer types are retained)

### Parallel Opportunities
- All test file updates (T014-T018) can proceed in parallel since they are independent files

### Dependencies
- Depends on WP01, WP02, WP03 (all source changes must be complete before tests can compile).

### Risks & Mitigations
- Composite state test assertions are the most complex to migrate due to `Children` type change from `SmcatDocument option` to `StateNode list`
- Generator tests need significant restructuring since the API changes from `string` return to `Result<StatechartDocument, GeneratorError>`

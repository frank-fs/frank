# Work Packages: ALPS Parser and Generator

**Inputs**: Design documents from `/kitty-specs/011-alps-parser-generator/`
**Prerequisites**: plan.md (required), spec.md (user stories), research.md, data-model.md
**GitHub Issue**: #97
**Dependencies**: Shared Statechart AST (spec 020) -- mapper WP only

**Tests**: Tests are included in each work package as they validate correctness of parsers and generators. Spec explicitly requires golden file tests and roundtrip validation.

**Organization**: Fine-grained subtasks (`Txxx`) roll up into work packages (`WPxx`). Each work package is independently deliverable and testable. WP02 and WP03 can be parallelized. WP05 is blocked on spec 020.

**Prompt Files**: Each work package references a matching prompt file in `tasks/` generated by `/spec-kitty.tasks`.

## Subtask Format: `[Txxx] [P?] Description`
- **[P]** indicates the subtask can proceed in parallel (different files/components).
- Paths are relative to repository root.

## Work Package WP01: Foundation -- ALPS Types, Golden Files, and Project Wiring (Priority: P0)

**Goal**: Define the ALPS-specific AST types, create golden file test constants for tic-tac-toe and onboarding examples, wire up project files, and establish basic type construction tests.
**Independent Test**: `dotnet build` succeeds for Frank.Statecharts (multi-target); TypeTests pass confirming AST structural equality and default type behavior.
**Prompt**: `tasks/WP01-foundation-types-golden-files.md`
**Estimated Prompt Size**: ~450 lines

### Included Subtasks
- [ ] T001 Define ALPS AST types in `src/Frank.Statecharts/Alps/Types.fs` (AlpsDocument, Descriptor, DescriptorType, AlpsDocumentation, AlpsExtension, AlpsLink, AlpsParseError, AlpsSourcePosition)
- [ ] T002 Update `src/Frank.Statecharts/Frank.Statecharts.fsproj` to include `Alps/Types.fs` in compile order (after Wsd files, before Types.fs)
- [ ] T003 [P] Create tic-tac-toe ALPS JSON golden file as string constant in `test/Frank.Statecharts.Tests/Alps/GoldenFiles.fs`
- [ ] T004 [P] Create tic-tac-toe ALPS XML golden file as string constant in `test/Frank.Statecharts.Tests/Alps/GoldenFiles.fs`
- [ ] T005 [P] Create onboarding ALPS golden files (JSON and XML) as string constants in `test/Frank.Statecharts.Tests/Alps/GoldenFiles.fs`
- [ ] T006 Create `test/Frank.Statecharts.Tests/Alps/TypeTests.fs` with AST construction, structural equality, and default descriptor type tests
- [ ] T007 Update `test/Frank.Statecharts.Tests/Frank.Statecharts.Tests.fsproj` to include all Alps test files in compile order

### Implementation Notes
- ALPS AST types are self-contained (AD-001) -- no dependency on spec 020 shared AST.
- Follow WSD pattern: types in `Alps/` subdirectory, module marked `internal`.
- Golden files must cover: all 4 descriptor types, nested descriptors, href references, rt values, ext elements, doc elements, link elements.
- Tic-tac-toe: States (XTurn, OTurn, Won, Draw), data elements (gameState, position, player), transitions (makeMove unsafe, viewGame safe), guards via ext.
- Onboarding: Linear flow (Welcome -> CollectEmail -> CollectProfile -> Review -> Complete), mix of safe and unsafe transitions.
- fsproj compile order: Alps/Types.fs after Wsd/Parser.fs, before Types.fs. Test files: GoldenFiles.fs first, then TypeTests.fs.

### Parallel Opportunities
- T003, T004, T005 (golden file creation) can proceed in parallel once GoldenFiles.fs module is established.
- T006 (TypeTests) can proceed once T001 (Types) is done.

### Dependencies
- None (starting package).

### Risks & Mitigations
- Golden file content accuracy: Ensure examples match ALPS spec structure. Cross-reference with research.md Research Area 1 and Area 4.
- fsproj ordering: F# compile order matters. Test with `dotnet build` after adding files.

## Work Package WP02: ALPS JSON Parser (Priority: P1) -- User Story 1

**Goal**: Implement the ALPS JSON parser that deserializes ALPS JSON documents into the typed ALPS AST, with comprehensive tests including golden file validation and edge cases.
**Independent Test**: Parse the tic-tac-toe and onboarding ALPS JSON golden files; verify every descriptor has correct id, type, nested descriptors, rt references, and ext elements.
**Prompt**: `tasks/WP02-json-parser.md`
**Estimated Prompt Size**: ~500 lines

### Included Subtasks
- [x] T008 Implement top-level parseAlpsJson function in `src/Frank.Statecharts/Alps/JsonParser.fs` -- parse root `alps` object, `version`, `doc` element
- [x] T009 Implement descriptor parsing in JsonParser.fs -- `id`, `type` (defaulting to Semantic), `href`, `rt`, nested `descriptor` array, `ext` array, `link` array, `doc` element
- [x] T010 Implement error handling in JsonParser.fs -- wrap `JsonDocument.Parse` in try/catch for `JsonException`, collect schema errors for missing fields, return `Result<AlpsDocument, AlpsParseError list>`
- [x] T011 Add `Alps/JsonParser.fs` to `Frank.Statecharts.fsproj` compile order (after Alps/Types.fs)
- [x] T012 Create `test/Frank.Statecharts.Tests/Alps/JsonParserTests.fs` -- tic-tac-toe golden file test, onboarding golden file test
- [x] T013 [P] Add edge case tests in JsonParserTests.fs -- empty document, missing type defaults to semantic, forward-compatible unknown properties, unicode characters, large documents
- [x] T014 [P] Add error case tests in JsonParserTests.fs -- malformed JSON, missing alps root, invalid structure

### Implementation Notes
- Use `System.Text.Json.JsonDocument` with `use` binding (constitution principle VI, R-003).
- Manual `JsonElement` traversal: `GetProperty`, `EnumerateArray`, `GetString`, `TryGetProperty` for optional fields.
- Forward-compatible: enumerate properties and skip unknowns (FR-017, AD-005).
- Return `Result<AlpsDocument, AlpsParseError list>` -- no exceptions for malformed input (AD-004).
- Pattern: `let parseAlpsJson (json: string) : Result<AlpsDocument, AlpsParseError list> = ...`

### Parallel Opportunities
- T013 (edge case tests) and T014 (error tests) can be written in parallel with T012 (golden file tests).
- WP02 and WP03 (XML parser) can proceed in parallel after WP01.

### Dependencies
- Depends on WP01 (needs Alps/Types.fs and GoldenFiles.fs).

### Risks & Mitigations
- JsonDocument is IDisposable -- must `use` it and copy all data before returning. Don't hold JsonElement references after disposal.
- Recursive descriptor parsing for nested descriptors -- ensure no stack overflow for deeply nested structures.

## Work Package WP03: ALPS XML Parser (Priority: P1) -- User Story 2

**Goal**: Implement the ALPS XML parser that deserializes ALPS XML documents into the same typed ALPS AST used by the JSON parser, with tests including golden file validation and cross-format equivalence verification.
**Independent Test**: Parse tic-tac-toe ALPS XML, compare resulting AST against the JSON-parsed AST, verify structural equivalence.
**Prompt**: `tasks/WP03-xml-parser.md`
**Estimated Prompt Size**: ~450 lines

### Included Subtasks
- [x] T015 Implement top-level parseAlpsXml function in `src/Frank.Statecharts/Alps/XmlParser.fs` -- parse root `<alps>` element, `version` attribute, `<doc>` child element
- [ ] T016 Implement descriptor parsing in XmlParser.fs -- `id`, `type`, `href`, `rt` as XML attributes; nested `<descriptor>`, `<ext>`, `<link>`, `<doc>` as child elements
- [ ] T017 Implement error handling in XmlParser.fs -- wrap `XDocument.Parse` in try/catch for `XmlException` with line/column info, schema validation, return `Result<AlpsDocument, AlpsParseError list>`
- [ ] T018 Add `Alps/XmlParser.fs` to `Frank.Statecharts.fsproj` compile order (after Alps/JsonParser.fs)
- [ ] T019 Create `test/Frank.Statecharts.Tests/Alps/XmlParserTests.fs` -- tic-tac-toe golden file test, cross-format equivalence test (JSON AST vs XML AST)
- [ ] T020 [P] Add edge case and error tests in XmlParserTests.fs -- empty document, namespace handling, forward-compatible unknown elements, malformed XML, missing attributes

### Implementation Notes
- Use `System.Xml.Linq.XDocument` (R-004). NOT IDisposable -- no `use` binding needed.
- ALPS XML rules: `alps`, `doc`, `descriptor`, `ext` are elements. All other properties are attributes (id, type, href, rt, rel, version, format, value).
- `<doc>` text content is the element's inner text. `format` is an attribute on `<doc>`.
- `XDocument.Parse` throws `XmlException` with `LineNumber`/`LinePosition` -- capture for `AlpsSourcePosition`.
- Must produce identical `AlpsDocument` as JSON parser for equivalent input (FR-002).

### Parallel Opportunities
- WP03 can proceed in parallel with WP02 (both depend only on WP01).
- T020 (edge/error tests) can be written in parallel with T019 (golden file tests).

### Dependencies
- Depends on WP01 (needs Alps/Types.fs and GoldenFiles.fs).

### Risks & Mitigations
- XML namespace declarations: Ensure the parser handles documents with or without namespace declarations (edge case from spec).
- Cross-format equivalence: The comparison test requires WP02's JSON parser to be available. If WP03 is built in parallel with WP02, the cross-format equivalence test can be deferred to integration or validated manually.

## Work Package WP04: ALPS JSON Generator and Roundtrip Tests (Priority: P2) -- User Stories 4, 5, 6

**Goal**: Implement the ALPS JSON generator that produces well-formed ALPS JSON from an AlpsDocument, plus roundtrip consistency tests that validate parse -> generate -> re-parse preserves all ALPS-expressible information.
**Independent Test**: Construct StateMachineMetadata matching tic-tac-toe, generate ALPS JSON, verify output matches golden file. Roundtrip: parse golden file, generate, re-parse, compare ASTs.
**Prompt**: `tasks/WP04-json-generator-roundtrip.md`
**Estimated Prompt Size**: ~500 lines

### Included Subtasks
- [ ] T021 Implement generateAlpsJson function in `src/Frank.Statecharts/Alps/JsonGenerator.fs` -- Utf8JsonWriter setup, root `alps` object, `version`, `doc` element
- [ ] T022 Implement descriptor generation in JsonGenerator.fs -- write semantic descriptors with nested children, transition descriptors with `type`, `rt`, `href`, `ext` arrays, `link` arrays, `doc` elements
- [ ] T023 Add `Alps/JsonGenerator.fs` to `Frank.Statecharts.fsproj` compile order (after Alps/XmlParser.fs, before Alps/Mapper.fs)
- [ ] T024 Create `test/Frank.Statecharts.Tests/Alps/JsonGeneratorTests.fs` -- tic-tac-toe golden file comparison, onboarding golden file comparison
- [ ] T025 [P] Add well-formedness and structural validation tests in JsonGeneratorTests.fs -- valid ALPS root structure, empty document generation, conditional transition (multiple rt) generation
- [ ] T026 Create `test/Frank.Statecharts.Tests/Alps/RoundTripTests.fs` -- parse ALPS JSON -> generate -> re-parse, compare ASTs for tic-tac-toe and onboarding
- [ ] T027 [P] Add cross-format roundtrip test in RoundTripTests.fs -- parse ALPS XML -> generate ALPS JSON -> re-parse JSON, verify equivalence

### Implementation Notes
- Use `Utf8JsonWriter` with `use` binding for both MemoryStream and writer (R-005, constitution principle VI).
- `JsonWriterOptions(Indented = true)` for human-readable output.
- Write descriptors recursively: each descriptor may contain nested child descriptors.
- Only write optional fields when they have values (version, doc, href, rt, ext, link).
- Roundtrip test pattern: `parseAlpsJson goldenJson |> Result.map generateAlpsJson |> Result.bind parseAlpsJson` then compare.
- Golden file comparison: structural match (same descriptor ids, types, rt, ext) rather than exact string match (JSON property order may vary).

### Parallel Opportunities
- T025 (well-formedness tests) and T027 (cross-format roundtrip) can proceed in parallel with T024 and T026.

### Dependencies
- Depends on WP02 (needs JsonParser for roundtrip tests and golden file validation).
- Cross-format roundtrip (T027) also needs WP03 (XML parser).

### Risks & Mitigations
- Golden file comparison must be structural, not string-exact. JSON property ordering may differ from golden file. Consider comparing parsed ASTs rather than raw strings.
- Utf8JsonWriter is IDisposable and must be flushed before reading the MemoryStream.

## Work Package WP05: ALPS Mapper -- Bidirectional Shared AST Mapping (Priority: P1, Blocked on Spec 020) -- User Story 3

**Goal**: Implement bidirectional mapping between the ALPS-specific AST and the shared statechart AST (StatechartDocument from spec 020), with tests for state extraction, transition mapping, guard extraction, and HTTP method hints.
**Independent Test**: Parse tic-tac-toe ALPS golden file, map to StatechartDocument, verify states (XTurn, OTurn, Won, Draw), transitions (makeMove, viewGame), and guard labels are present.
**Prompt**: `tasks/WP05-mapper.md`
**Estimated Prompt Size**: ~400 lines
**Blocked On**: Spec 020 (shared statechart AST) must land before this WP can begin.

### Included Subtasks
- [ ] T028 Implement `toStatechartDocument` in `src/Frank.Statecharts/Alps/Mapper.fs` -- map semantic descriptors to StateNodes, transition descriptors to TransitionEdges, extract guards from ext elements, derive HTTP method hints
- [ ] T029 Implement `fromStatechartDocument` in `src/Frank.Statecharts/Alps/Mapper.fs` -- map StateNodes to semantic descriptors, TransitionEdges to transition descriptors with correct ALPS types, generate ext elements for guards
- [ ] T030 Add `Alps/Mapper.fs` to `Frank.Statecharts.fsproj` compile order (last in Alps group, after Alps/JsonGenerator.fs)
- [ ] T031 Create `test/Frank.Statecharts.Tests/Alps/MapperTests.fs` -- state extraction, transition mapping with rt targets, guard label extraction from ext, HTTP method hints (safe->GET, unsafe->POST, idempotent->PUT), empty/minimal document handling

### Implementation Notes
- Mapper is the ONLY component depending on spec 020's shared AST types (AD-001, R-006).
- State mapping: semantic descriptors containing transition hrefs become states with available transitions.
- Transition mapping: descriptor `id` -> event name, parent semantic descriptor -> source state, `rt` value -> target state, `type` -> AlpsAnnotation.
- Guard extraction: ext elements with `id = "guard"` -> guard expression on TransitionEdge.
- HTTP method mapping (FR-010): Safe -> GET, Unsafe -> POST, Idempotent -> PUT (with documented DELETE caveat).
- Missing ALPS info: no initial state, no workflow ordering -- leave those fields empty/default in StatechartDocument.
- Use `AlpsMeta` annotation stub from spec 020: `AlpsTransitionType`, `AlpsDescriptorHref`, `AlpsExtension`.

### Parallel Opportunities
- None within this WP (Mapper.fs and MapperTests.fs are tightly coupled).
- This WP is independent of WP04 (generator) but blocked on spec 020.

### Dependencies
- Depends on WP01 (ALPS types).
- Depends on WP02 (JSON parser, for test golden file parsing).
- **Blocked on spec 020** (shared statechart AST types must exist before implementation).

### Risks & Mitigations
- Spec 020 `AlpsMeta` stub may be insufficient -- per R-007, accept as-is and propose amendments if gaps found.
- Lossy mapping: ALPS cannot express workflow ordering or initial state. Document gaps clearly in mapper output.
- Conditional transitions: One descriptor per rt value means multiple TransitionEdges for the same event name. Mapper must handle this correctly.
